SSH and security issues:


-t rsa: specifies an algorithm to create the key (in this case rsa)

-b 4096: specifies (in bytes) mathematical complexity of the key (in this case 4096)

-C specifies email

if everything is OK you should get confirmation, a path to store your key, the possibility to add an extra password and a randomart image

At this moment 2 files have been created on the path stated, a public key (id_rsa.pub and your private key id_rsa)



If you get pid number everything is OK and your ssh is running


If you get pid number everything is OK and your ssh is running


If you changed path you should ssh-add + your actual path

Now it is time to share your public key with Github. please go to https://github.com/settings/ssh and create a new public ssh key. You should have one public key connected for every device you have, never share the same key between devices.

On this step you should copy the content of your public ssh key (id_rsa.pub stored in c/Users/PC name/.ssh by default) on the corresponding box (on Github web site).

You should never reveal the content of your private key (id_rsa) to anyone or submitting it to gitHub. If you do so, remove that key from your computer and Github and create a new one.
